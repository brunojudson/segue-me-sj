1. Visão Geral da Arquitetura

O Sistema Segue-me segue uma arquitetura em camadas baseada no padrão MVC (Model-View-Controller) com extensões para separação de responsabilidades através de camadas de serviço e persistência.

💡 Benefícios da Arquitetura em Camadas:
  • Separação de Responsabilidades: Cada camada possui função específica
  • Manutenibilidade: Alterações em uma camada não afetam as demais
  • Testabilidade: Camadas podem ser testadas isoladamente
  • Reusabilidade: Lógica de negócio pode ser reutilizada em diferentes interfaces
  • Escalabilidade: Facilita distribuição em múltiplos servidores

2. Camadas da Arquitetura

📱 Camada de Apresentação (View)

Tecnologia: JSF 2.3 + Facelets + PrimeFaces 12 + HTML5 + CSS3 + JavaScript

Responsabilidade: Interface com o usuário, renderização de páginas, validações client-side

Localização: src/main/webapp/pages/, src/main/webapp/templates/

Componentes:

  • Templates: Layout padrão (header, menu, footer, content)
  • Páginas: Formulários de cadastro, listas, visualizações detalhadas
  • Componentes Customizados: resources/components/
  • Recursos Estáticos: CSS, JS, imagens em resources/

Principais Bibliotecas:

  • PrimeFaces 12.0.0 - Componentes ricos (DataTable, Calendar, FileUpload, etc.)
  • PrimeFaces Extensions 12.0.0 - Componentes adicionais
  • OmniFaces 2.7.3 - Utilitários JSF (ValidateAll, ViewParam, etc.)
🎮 Camada de Controle (Controller)

Tecnologia: CDI Managed Beans (@Named, @ViewScoped / @SessionScoped)

Responsabilidade: Controlar fluxo de navegação, intermediar View e Service, manter estado da sessão

Localização: src/main/java/br/com/segueme/controller/

Padrão de Nomenclatura:

  • *Controller.java - Ex: PessoaController, EncontroController
  • Anotação @Named para injeção no JSF
  • Escopo @ViewScoped para formulários
  • Escopo @SessionScoped para dados de sessão (ex: LoginController)

Responsabilidades Típicas:

  • Receber dados do formulário (binding com XHTML)
  • Validar entrada do usuário
  • Chamar métodos da camada de serviço
  • Gerenciar mensagens de sucesso/erro (FacesMessage)
  • Controlar navegação entre páginas
@Named @ViewScoped public class PessoaController implements Serializable { @Inject private PessoaService pessoaService; private Pessoa pessoa = new Pessoa(); private List<Pessoa> pessoas; public String salvar() { try { pessoaService.salvar(pessoa); FacesUtil.addSuccessMessage("Pessoa salva com sucesso!"); limpar(); return "lista?faces-redirect=true"; } catch (Exception e) { FacesUtil.addErrorMessage("Erro ao salvar: " + e.getMessage()); return null; } } }
⚙️ Camada de Serviço (Service)

Tecnologia: CDI (@ApplicationScoped) + Transações JTA (@Transactional)

Responsabilidade: Implementar regras de negócio, orquestrar operações, gerenciar transações

Localização: src/main/java/br/com/segueme/service/

Padrão de Nomenclatura:

  • *Service.java - Ex: PessoaService, EncontroService
  • Anotação @ApplicationScoped (singleton)

Responsabilidades:

  • Validações de negócio complexas
  • Orquestração de múltiplas entidades
  • Cálculos e processamentos
  • Controle transacional (@Transactional)
  • Delegação para repositórios
@ApplicationScoped public class EncontristaService implements Serializable { @Inject private EntityManager em; @Inject private PessoaService pessoaService; @Transactional public void inscrever(Encontrista encontrista) { // Validação: pessoa já inscrita? if (jaInscrito(encontrista.getPessoa(), encontrista.getEncontro())) { throw new NegocioException("Pessoa já inscrita neste encontro"); } // Calcular idade encontrista.setIdade(calcularIdade(encontrista.getPessoa())); // Gerar token único encontrista.setTokenFicha(UUID.randomUUID().toString()); // Persistir em.persist(encontrista); } }
💾 Camada de Persistência (Repository)

Tecnologia: JPA 2.2 + EntityManager + JPQL

Responsabilidade: Acesso ao banco de dados, queries, operações CRUD

Localização: src/main/java/br/com/segueme/repository/

Padrão:

  • Interface *Repository.java
  • Implementação *RepositoryImpl.java
  • Injeção de EntityManager

Operações Comuns:

  • CRUD básico (create, read, update, delete)
  • Queries customizadas (JPQL, Criteria API)
  • Paginação e ordenação
  • Buscas com filtros dinâmicos
public interface PessoaRepository { void salvar(Pessoa pessoa); Pessoa buscarPorId(Long id); List<Pessoa> listarTodos(); List<Pessoa> buscarPorNome(String nome); void remover(Pessoa pessoa); } @ApplicationScoped public class PessoaRepositoryImpl implements PessoaRepository { @PersistenceContext private EntityManager em; @Override public void salvar(Pessoa pessoa) { if (pessoa.getId() == null) { em.persist(pessoa); } else { em.merge(pessoa); } } @Override public List<Pessoa> buscarPorNome(String nome) { return em.createQuery( "SELECT p FROM Pessoa p WHERE LOWER(p.nome) LIKE :nome", Pessoa.class) .setParameter("nome", "%" + nome.toLowerCase() + "%") .getResultList(); } }
🗃️ Camada de Dados (Model/Entity)

Tecnologia: JPA 2.2 + Hibernate 5.6 + Bean Validation

Responsabilidade: Representar modelo de domínio, mapeamento objeto-relacional

Localização: src/main/java/br/com/segueme/entity/

Padrões:

  • Anotação @Entity
  • Lombok para reduzir boilerplate (@Data, @NoArgsConstructor)
  • Bean Validation (@NotNull, @Size, @Pattern, etc.)
  • Serializable para suporte a sessão HTTP
@Entity @Table(name = "pessoa") @Data @NoArgsConstructor public class Pessoa implements Serializable { @Id @GeneratedValue(strategy = GenerationType.IDENTITY) private Long id; @NotBlank(message = "Nome é obrigatório") @Size(min = 3, max = 100) @Column(nullable = false, length = 100) private String nome; @Column(unique = true, length = 14) private String cpf; @NotNull @Past @Column(name = "data_nascimento", nullable = false) private LocalDate dataNascimento; @Pattern(regexp = "\\(\\d{2}\\) \\d{4,5}-\\d{4}") @Column(unique = true, length = 15) private String telefone; @Email @Column(length = 100) private String email; // ... demais atributos }

3. Módulos Transversais

3.1. Segurança e Autenticação

  • Filtro de Autenticação: AuthFilter.java - Intercepta requisições e valida sessão
  • Hash de Senhas: BCrypt via jbcrypt
  • Controle de Sessão: LoginController gerencia login/logout
  • Permissões: Entidade Permissao com níveis de acesso

3.2. Auditoria

  • Service: AuditoriaService registra ações críticas
  • Entidade: Auditoria armazena logs
  • Dados Registrados: Usuário, data/hora, ação, entidade afetada, detalhes

3.3. API REST

Localização: src/main/java/br/com/segueme/api/

  • Tecnologia: JAX-RS (Jakarta REST API)
  • Application: RestApplication - Base path /api
  • Resources:
    • PessoaResource - /api/pessoas
    • EncontroResource - /api/encontros
    • EncontristaResource - /api/encontristas
    • EquipeResource - /api/equipes
  • CORS: CorsFilter habilita cross-origin para consumo externo

3.4. Relatórios

  • Service: RelatorioService - Geração de relatórios
  • PDF: PdfService com iText / JasperReports
  • QR Code: QrCodeService com ZXing library
  • Fichas Digitais: PDFs com dados + foto + QR Code

3.5. Utilitários

Localização: src/main/java/br/com/segueme/util/

  • calculaIdade.java - Cálculo de idade
  • GerarHashSenha.java - Utilitário para hash BCrypt
  • BackupPostgres.java - Backup automatizado do banco
  • RestorePostgres.java - Restauração de backup
  • DatabaseInitializer.java - Inicialização de dados

3.6. Conversores

Localização: src/main/java/br/com/segueme/converter/

  • EncontroConverter - Conversão de ID para objeto Encontro no JSF
  • EquipeConverter - Conversão de ID para objeto Equipe
  • LocalDateConverter - Conversão entre String ↔ LocalDate
  • LocalDateTimeConverter - Conversão entre String ↔ LocalDateTime
  • TemaPalestraConverter - Conversão de enum TemaPalestra

9. Boas Práticas Implementadas

9.1. Código Limpo

  • Nomenclatura descritiva e em português
  • Métodos pequenos e focados (Single Responsibility)
  • Uso de Lombok para reduzir boilerplate
  • Separação clara de responsabilidades entre camadas

9.2. Segurança

  • Senhas hashadas com BCrypt (nunca plain text)
  • Validação server-side obrigatória
  • Prepared Statements via JPA (proteção contra SQL Injection)
  • Filtro de autenticação em todas as páginas protegidas

9.3. Performance

  • FetchType.LAZY em relacionamentos @OneToMany e @ManyToMany
  • Paginação em listas grandes (DataTable + LazyDataModel)
  • Índices em campos de busca frequente
  • Cache de segundo nível do Hibernate (quando necessário)

9.4. Manutenibilidade

  • Arquitetura em camadas bem definida
  • Injeção de dependências (fácil mockar em testes)
  • Configurações externalizadas
  • Logging adequado com Logback
